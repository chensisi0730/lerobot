# 从 Hub 加载环境

**EnvHub** 功能允许你通过一行代码直接从 Hugging Face Hub 加载仿真环境。这解锁了一种强大的新协作模式：环境不再被锁定在单一的库中，任何人都可以发布自定义环境并与社区分享。

## 什么是 EnvHub？

EnvHub 让你能够使用自己的机器人模型和场景创建自定义机器人仿真环境，并通过 LeRobot 框架让任何人都能轻松使用。

EnvHub 包存储在 Hugging Face Hub 上，可以通过 LeRobot 在你的 AI 机器人项目中无缝拉取和使用，只需一行代码。

借助 EnvHub，你可以：

1. **创建和发布环境**到 Hugging Face Hub 作为 Git 仓库，无需打包麻烦即可分发复杂物理仿真
2. **动态加载环境**，无需将其安装为包
3. **使用 Git 语义对环境变化进行版本控制和跟踪**
4. **发现**社区分享的新仿真任务

这种设计意味着你可以从在 Hub 上发现有趣的环境到在几秒钟内运行实验，或者创建自己的自定义机器人和环境，而无需担心依赖冲突或复杂的安装程序。

当你创建 EnvHub 包时，你可以在其中构建任何你想要的东西，并使用任何你喜欢的仿真工具：这是你自己的playground。唯一的要求是包包含一个定义环境的 `env.py` 文件，让 LeRobot 能够加载和使用你的 EnvHub 包。

这个 `env.py` 文件需要暴露一个小 API，以便 LeRobot 可以加载和运行它。特别地，你必须提供一个 `make_env(n_envs: int = 1, use_async_envs: bool = False)` 或 `make_env(n_envs: int = 1, use_async_envs: bool = False, cfg: EnvConfig)` 函数，这是 LeRobot 的主要入口点。它应该返回以下之一：

- 一个 `gym.vector.VectorEnv`（最常见）
- 一个单一的 `gym.Env`（将自动包装）
- 一个字典映射 `{suite_name: {task_id: VectorEnv}}`（用于多任务基准）

你也可以向 `make_env` 传递一个 `EnvConfig` 对象来配置环境（例如环境数量、任务、相机名称、初始状态、控制模式、episode 长度等）。

最后，你的环境必须实现标准的 `gym.vector.VectorEnv` 接口，以便它能与 LeRobot 一起工作，包括 `reset` 和 `step` 等方法。

## 快速开始

从 Hub 加载环境非常简单：

```python
from lerobot.envs import make_env

# 加载 hub 环境（需要明确同意运行远程代码）
env = make_env("lerobot/cartpole-env", trust_remote_code=True)
```

> **安全提示**：从 Hub 加载环境会执行来自第三方仓库的 Python 代码。只在你信任的仓库使用 `trust_remote_code=True`。我们强烈建议锁定到特定的提交哈希以确保可重现性和安全性。

## 仓库结构

要使你的环境能够从 Hub 加载，你的仓库必须至少包含：

### 必需文件

**`env.py`**（或自定义 Python 文件）

- 必须暴露一个 `make_env(n_envs: int, use_async_envs: bool)` 函数
- 此函数应返回以下之一：
  - 一个 `gym.vector.VectorEnv`（最常见）
  - 一个单一的 `gym.Env`（将自动包装）
  - 一个字典映射 `{suite_name: {task_id: VectorEnv}}`（用于多任务基准）

### 可选文件

**`requirements.txt`**

- 列出你的环境需要的任何额外依赖
- 用户需要在加载你的环境之前手动安装这些

**`README.md`**

- 记录你的环境：它实现的任务、观测/动作空间、奖励等
- 包含使用示例和任何特殊设置说明

**`.gitignore`**

- 从仓库中排除不必要的文件

### 示例仓库结构

```

    └── task_config.yaml
```

## 创建你的环境仓库

### 第一步：定义你的环境

创建一个带有 `make_env` 函数的 `env.py` 文件：

```python
# env.py
import gymnasium as gym

def make_env(n_envs: int = 1, use_async_envs: bool = False):
    """
    为你的自定义任务创建向量化环境。

    参数:
        n_envs: 并行环境数量
        use_async_envs: 是否使用 AsyncVectorEnv 或 SyncVectorEnv

    返回:
        gym.vector.VectorEnv 或将 suite 名称映射到向量化环境的字典
    """
    def _make_single_env():
        # 创建你的自定义环境
        return gym.make("CartPole-v1")

    # 选择向量化环境类型
    env_cls = gym.vector.AsyncVectorEnv if use_async_envs else gym.vector.SyncVectorEnv

    # 创建向量化环境
    vec_env = env_cls([_make_single_env for _ in range(n_envs)])

    return vec_env
```

### 第二步：本地测试

上传之前，在本地测试你的环境：

```python
from lerobot.envs.utils import _load_module_from_path, _call_make_env, _normalize_hub_result

# 加载你的模块
module = _load_module_from_path("./env.py")

# 测试 make_env 函数
result = _call_make_env(module, n_envs=2, use_async_envs=False)
normalized = _normalize_hub_result(result)

# 验证它能正常工作
suite_name = next(iter(normalized))
env = normalized[suite_name][0]
obs, info = env.reset()
print(f"观测形状: {obs.shape if hasattr(obs, 'shape') else type(obs)}")
env.close()
```

## 从 Hub 加载环境

### 基本用法

```python
from lerobot.envs import make_env

# 从 hub 加载
envs_dict = make_env(
    "username/my-custom-env",
    n_envs=4,
    trust_remote_code=True
)

# 访问环境
suite_name = next(iter(envs_dict))
env = envs_dict[suite_name][0]

# 像使用普通 gym 环境一样使用它
obs, info = env.reset()
action = env.action_space.sample()
obs, reward, terminated, truncated, info = env.step(action)
```

### 高级：锁定到特定版本

为确保可重现性和安全性，锁定到特定的 Git 版本：

```python
# 锁定到特定分支
env = make_env("username/my-env@main", trust_remote_code=True)

# 锁定到特定提交（推荐用于论文/实验）
env = make_env("username/my-env@abc123def456", trust_remote_code=True)

# 锁定到标签
env = make_env("username/my-env@v1.0.0", trust_remote_code=True)
```

### 自定义文件路径

如果你的环境定义不在 `env.py` 中：

```python
# 从自定义文件加载
env = make_env("username/my-env:custom_env.py", trust_remote_code=True)

# 结合版本锁定
env = make_env("username/my-env@v1.0:envs/task_a.py", trust_remote_code=True)
```

### 异步环境

为获得更好的多环境性能：

```python
envs_dict = make_env(
    "username/my-env",
    n_envs=8,
    use_async_envs=True,  # 使用 AsyncVectorEnv 进行并行执行
    trust_remote_code=True
)
```

## URL 格式参考

hub URL 格式支持多种模式：

| 模式 | 描述 | 示例 |
| -------------------- | ------------------------------ | -------------------------------------- |
| `user/repo`          | 从 main 分支加载 `env.py` | `make_env("lerobot/pusht-env")`        |
| `user/repo@revision` | 从特定版本加载    | `make_env("lerobot/pusht-env@main")`   |
| `user/repo:path`     | 加载自定义文件               | `make_env("lerobot/envs:pusht.py")`    |
| `user/repo@rev:path` | 版本 + 自定义文件         | `make_env("lerobot/envs@v1:pusht.py")` |

## 多任务环境

对于多任务基准测试（如 LIBERO），返回嵌套字典：

```python
def make_env(n_envs: int = 1, use_async_envs: bool = False):
    env_cls = gym.vector.AsyncVectorEnv if use_async_envs else gym.vector.SyncVectorEnv

    # 返回字典: {suite_name: {task_id: VectorEnv}}
    return {
        "suite_1": {
            0: env_cls([lambda: gym.make("Task1-v0") for _ in range(n_envs)]),
            1: env_cls([lambda: gym.make("Task2-v0") for _ in range(n_envs)]),
        },
        "suite_2": {
            0: env_cls([lambda: gym.make("Task3-v0") for _ in range(n_envs)]),
        }
    }
```

## 安全注意事项

> **重要**：`trust_remote_code=True` 标志是执行来自 Hub 的环境代码所必需的。这是出于安全考虑的设计。

从 Hub 加载环境时：

1. **首先审查代码**：加载之前访问仓库并检查 `env.py`
2. **锁定到提交**：使用特定的提交哈希以确保可重现性
3. **检查依赖项**：查看 `requirements.txt` 中是否有可疑的包
4. **使用可信来源**：优先选择官方组织或知名研究人员
5. **如需要隔离运行**：在隔离环境（容器、VM）中运行不受信任的代码

安全使用示例：

```python
# ❌ 不好：未检查就加载
env = make_env("random-user/untrusted-env", trust_remote_code=True)

# ✅ 好：审查代码，然后锁定到特定提交
# 1. 访问 https://huggingface.co/trusted-org/verified-env
# 2. 审查 env.py 文件
# 3. 复制提交哈希
env = make_env("trusted-org/verified-env@a1b2c3d4", trust_remote_code=True)
```

## 示例：从 Hub 加载 CartPole

以下是使用参考 CartPole 环境的完整示例：

```python
from lerobot.envs import make_env
import numpy as np

# 加载环境
envs_dict = make_env("lerobot/cartpole-env", n_envs=4, trust_remote_code=True)

# 获取向量化环境
suite_name = next(iter(envs_dict))
env = envs_dict[suite_name][0]

# 运行一个简单的 episode
obs, info = env.reset()
done = np.zeros(env.num_envs, dtype=bool)
total_reward = np.zeros(env.num_envs)

while not done.all():
    # 随机策略
    action = env.action_space.sample()
    obs, reward, terminated, truncated, info = env.step(action)
    total_reward += reward
    done = terminated | truncated

print(f"平均奖励: {total_reward.mean():.2f}")
env.close()
```

## 故障排除

### "Refusing to execute remote code"

你必须明确传递 `trust_remote_code=True`：

```python
env = make_env("user/repo", trust_remote_code=True)
```

### "Module X not found"

hub 环境有你需要安装的依赖：

```bash
# 检查仓库的 requirements.txt 并安装依赖
pip install gymnasium numpy
```

### "make_env not found in module"

你的 `env.py` 必须暴露 `make_env` 函数：

```python
def make_env(n_envs: int, use_async_envs: bool):
    # 你的实现
    pass
```

### 环境返回错误类型

`make_env` 函数必须返回：

- 一个 `gym.vector.VectorEnv`，或
- 一个单一的 `gym.Env`，或
- 一个字典 `{suite_name: {task_id: VectorEnv}}`

## 最佳实践

1. **记录你的环境**：在 README 中包含观测/动作空间描述、奖励结构和终止条件
2. **添加 requirements.txt**：列出所有带版本的依赖项
3. **彻底测试**：推送前在本地验证你的环境能正常工作
4. **使用语义版本控制**：用版本号标记发布
5. **添加示例**：在 README 中包含使用示例
6. **保持简单**：尽可能减少依赖项
7. **为你的工作添加许可证**：添加 LICENSE 文件以明确使用条款
