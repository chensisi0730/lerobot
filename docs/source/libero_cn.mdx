# LIBERO

LIBERO 是一个专为研究**机器人终身学习**而设计的基准——这个概念认为机器人需要随着时间推移不断学习和适应用户，而不仅仅是一次预训练。它提供了一组标准化的操作任务，专注于**知识迁移**：机器人如何将已学到的知识应用到新情境中。通过在 LIBERO 上评估，不同的算法可以公平比较，研究人员可以相互借鉴工作。

- 论文：[Benchmarking Knowledge Transfer for Lifelong Robot Learning](https://arxiv.org/abs/2306.03310)
- GitHub：[Lifelong-Robot-Learning/LIBERO](https://github.com/Lifelong-Robot-Learning/LIBERO)
- 项目网站：[libero-project.github.io](https://libero-project.github.io)

![LIBERO 基准概述](https://libero-project.github.io/assets/img/libero/fig1.png)

## 可用任务

LIBERO 包含**五个任务套件**，涵盖**130 个任务**，从简单的物体操作到复杂的多步骤场景：

| 套件 | CLI 名称 | 任务数 | 描述 |
| -------------- | ---------------- | ----- | -------------------------------------------------- |
| LIBERO-Spatial | `libero_spatial` | 10    | 需要空间关系推理的任务 |
| LIBERO-Object  | `libero_object`  | 10    | 围绕操作不同物体的任务 |
| LIBERO-Goal    | `libero_goal`    | 10    | 目标条件任务，目标是变化的 |
| LIBERO-90      | `libero_90`      | 90    | 来自 LIBERO-100 集合的短视界任务 |
| LIBERO-Long    | `libero_10`      | 10    | 来自 LIBERO-100 集合的长视界任务 |

## 安装

按照 LeRobot 安装说明后：

```bash
pip install -e ".[libero]"
```

<Tip>
LIBERO 需要 Linux（`sys_platform == 'linux'`）。LeRobot 使用 MuJoCo 进行仿真——在训练或评估前设置渲染后端：

```bash
export MUJOCO_GL=egl  # 用于无头服务器（HPC、云）
```

</Tip>

## 评估

### 默认评估（推荐）

在四个标准套件上评估（每个任务 10 个 episode）：

```bash
lerobot-eval \
  --policy.path="your-policy-id" \
  --env.type=libero \
  --env.task=libero_spatial,libero_object,libero_goal,libero_10 \
  --eval.batch_size=1 \
  --eval.n_episodes=10 \
  --env.max_parallel_tasks=1
```

### 单套件评估

在单个 LIBERO 套件上评估：

```bash
lerobot-eval \
  --policy.path="your-policy-id" \
  --env.type=libero \
  --env.task=libero_object \
  --eval.batch_size=2 \
  --eval.n_episodes=3
```

- `--env.task` 选择套件（`libero_object`、`libero_spatial` 等）。
- `--env.task_ids` 限制特定任务索引（`[0]`、`[1,2,3]` 等）。省略则在套件中运行所有任务。
- `--eval.batch_size` 控制多少环境并行运行。
- `--eval.n_episodes` 设置每个任务运行多少个 episode。

### 多套件评估

通过传递逗号分隔的列表一次在多个套件上对策略进行基准测试：

```bash
lerobot-eval \
  --policy.path="your-policy-id" \
  --env.type=libero \
  --env.task=libero_object,libero_spatial \
  --eval.batch_size=1 \
  --eval.n_episodes=2
```

### 控制模式

LIBERO 支持两种控制模式——`relative`（默认）和 `absolute`。不同的 VLA 检查点使用不同的动作参数化进行训练，因此请确保模式与你的策略匹配：

```bash
--env.control_mode=relative   # 或 "absolute"
```

### 策略输入和输出

**观测：**

- `observation.state` — 8 维本体感受特征（末端执行器位置、轴角方向、夹爪 qpos）
- `observation.images.image` — 主相机视角（`agentview_image`），HWC uint8
- `observation.images.image2` — 腕部相机视角（`robot0_eye_in_hand_image`），HWC uint8

<Tip warning={true}>
  LeRobot 强制要求视觉特征使用 `.images.*` 前缀。确保你的策略配置 `input_features` 使用相同的命名键，并且你的数据集元数据键遵循此约定。如果你的数据包含不同的键，你必须重命名观测以匹配策略期望，因为命名键被编码在归一化统计层中。
</Tip>

**动作：**

- `Box(-1, 1, shape=(7,))` 中的连续控制 — 6D 末端执行器增量 + 1D 夹爪

### 推荐的评估 episode 数

为获得可重现的基准测试，在所有四个标准套件（Spatial、Object、Goal、Long）上每个任务使用 **10 个 episode**。这总共给出 400 个 episode，与发布结果所使用的协议相匹配。

## 训练

### 数据集

我们提供了一个与 LeRobot 完全兼容的预处理 LIBERO 数据集：

- [HuggingFaceVLA/libero](https://huggingface.co/datasets/HuggingFaceVLA/libero)

作为参考，Physical Intelligence 发布的原始数据集：

- [physical-intelligence/libero](https://huggingface.co/datasets/physical-intelligence/libero)

### 示例训练命令

```bash
lerobot-train \
  --policy.type=smolvla \
  --policy.repo_id=${HF_USER}/libero-test \
  --policy.load_vlm_weights=true \
  --dataset.repo_id=HuggingFaceVLA/libero \
  --env.type=libero \
  --env.task=libero_10 \
  --output_dir=./outputs/ \
  --steps=100000 \
  --batch_size=4 \
  --eval.batch_size=1 \
  --eval.n_episodes=1 \
  --eval_freq=1000
```

## 复现发布的结果

我们在 LIBERO 基准上复现了 Pi0.5 的结果。我们采用 Physical Intelligence 的 LIBERO 基础模型（`pi05_libero`）并使用 [HuggingFace LIBERO 数据集](https://huggingface.co/datasets/HuggingFaceVLA/libero) 进行额外 6000 步的微调，使用 bf16，在 8 个 H100 GPU 上批量大小为 256。

微调后的模型：[lerobot/pi05_libero_finetuned](https://huggingface.co/lerobot/pi05_libero_finetuned)

### 评估命令

```bash
lerobot-eval \
  --output_dir=./eval_logs/ \
  --env.type=libero \
  --env.task=libero_spatial,libero_object,libero_goal,libero_10 \
  --eval.batch_size=1 \
  --eval.n_episodes=10 \
  --policy.path=pi05_libero_finetuned \
  --policy.n_action_steps=10 \
  --env.max_parallel_tasks=1
```

我们设置 `n_action_steps=10`，与原始 OpenPI 实现相匹配。

### 结果

| 模型 | LIBERO Spatial | LIBERO Object | LIBERO Goal | LIBERO 10 | 平均 |
| ------------------- | -------------- | ------------- | ----------- | --------- | -------- |
| **Pi0.5 (LeRobot)** | 97.0           | 99.0          | 98.0        | 96.0      | **97.5** |

这些结果与 Physical Intelligence 报告的[原始结果](https://github.com/Physical-Intelligence/openpi/tree/main/examples/libero#results)一致：

| 模型 | LIBERO Spatial | LIBERO Object | LIBERO Goal | LIBERO 10 | 平均 |
| ------------------ | -------------- | ------------- | ----------- | --------- | --------- |
| **Pi0.5 (OpenPI)** | 98.8           | 98.2          | 98.0        | 92.4      | **96.85** |
