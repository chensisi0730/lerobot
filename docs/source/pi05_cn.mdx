# π₀.₅ (Pi05) 策略

π₀.₅ 是来自 Physical Intelligence 的**具有开放世界泛化能力的视觉-语言-动作模型**。LeRobot 实现改编自他们的开源 [OpenPI](https://github.com/Physical-Intelligence/openpi) 仓库。

## 模型概述

π₀.₅ 是由 [Physical Intelligence](https://www.physicalintelligence.company/blog/pi05) 开发的 π₀ 的重大演进，旨在解决机器人领域的一个重大挑战：**开放世界泛化**。虽然机器人在受控环境中可以执行令人印象深刻的任务，但 π₀.₅ 旨在泛化到训练中从未见过全新环境和情况。

### 泛化挑战

正如 Physical Intelligence 所解释的，根本挑战不是执行敏捷或灵巧的任务，而是泛化——在新设置中与新物体正确执行任务的能力。考虑一个清洁不同家庭的机器人：每个家庭都有不同位置的不同物体。泛化必须在多个层面发生：

- **物理层面**：理解如何拿起勺子（握把）或盘子（边缘），即使在杂乱环境中存在看不见的物体
- **语义层面**：理解任务语义，将衣服和鞋子放在哪里（洗衣篮，而不是床上），以及清理溢出物使用什么工具
- **环境层面**：适应"杂乱"的现实环境，如家庭、杂货店、办公室和医院

### 异构数据上的协同训练

π₀.₅ 的突破性创新是**在异构数据源上进行协同训练**。模型从以下来源学习：

1. **多模态网络数据**：图像 captioning、视觉问答、目标检测
2. **口头指令**：人类通过复杂任务逐步指导机器人
3. **子任务命令**：高级语义行为标签（例如，对于未整理的床，"拿起枕头"）
4. **跨本体机器人数据**：来自具有不同能力的各种机器人平台的数据
5. **多环境数据**：部署在许多不同家庭中的静态机器人
6. **移动操作数据**：约 400 小时的移动机器人演示

这种多样化的训练混合创造了一个"课程"，能够在物理、视觉和语义层面同时实现泛化。

## 使用方法

要在你的 LeRobot 配置中使用 π₀.₅，将策略类型指定为：

```python
policy.type=pi05
```

## 训练

### 训练命令示例

以下是使用你自己的数据集微调基础 π₀.₅ 模型的完整训练命令：

```bash
lerobot-train \
    --dataset.repo_id=your_dataset \
    --policy.type=pi05 \
    --output_dir=./outputs/pi05_training \
    --job_name=pi05_training \
    --policy.repo_id=your_repo_id \
    --policy.pretrained_path=lerobot/pi05_base \
    --policy.compile_model=true \
    --policy.gradient_checkpointing=true \
    --wandb.enable=true \
    --policy.dtype=bfloat16 \
    --policy.freeze_vision_encoder=false \
    --policy.train_expert_only=false \
    --steps=3000 \
    --policy.device=cuda \
    --batch_size=32
```

### 关键训练参数

- **`--policy.compile_model=true`**：启用模型编译以加快训练
- **`--policy.gradient_checkpointing=true`**：在训练期间显著减少内存使用
- **`--policy.dtype=bfloat16`**：使用混合精度训练以提高效率
- **`--batch_size=32`**：训练批量大小，根据你的 GPU 内存调整
- **`--policy.pretrained_path=lerobot/pi05_base`**：你想要微调的基础 π₀.₅ 模型，选项包括：
  - [lerobot/pi05_base](https://huggingface.co/lerobot/pi05_base)
  - [lerobot/pi05_libero](https://huggingface.co/lerobot/pi05_libero)（专门在 Libero 数据集上训练）

### 训练参数说明

| 参数 | 默认值 | 描述 |
| ----------------------- | ------- | ------------------------------------------- |
| `freeze_vision_encoder` | `false` | 不冻结视觉编码器 |
| `train_expert_only` | `false` | 不冻结 VLM，训练所有参数 |

**💡 提示**：设置 `train_expert_only=true` 会冻结 VLM，只训练动作专家和投影，允许以减少的内存使用进行微调。

如果你的数据集没有使用 `quantiles` 转换，你可以使用以下命令转换它：

```bash
python src/lerobot/datasets/v30/augment_dataset_quantile_stats.py \
    --repo-id=your_dataset
```

或者使用以下归一化映射训练 pi05：`--policy.normalization_mapping='{"ACTION": "MEAN_STD", "STATE": "MEAN_STD", "VISUAL": "IDENTITY"}'`

## 相对动作

默认情况下，π₀.₅ 预测绝对动作。你可以启用**相对动作**，让模型预测相对于当前机器人状态的偏移。这可以提高某些设置下的训练稳定性。

要使用相对动作，首先通过 CLI 在相对空间中重新计算你的数据集统计信息：

```bash
lerobot-edit-dataset \
    --repo_id your_dataset \
    --operation.type recompute_stats \
    --operation.relative_action true \
    --operation.chunk_size 50 \
    --operation.relative_exclude_joints "['gripper']" \
    --push_to_hub true
```

或等效的 Python 代码：

```python
from lerobot.datasets import LeRobotDataset, recompute_stats

dataset = LeRobotDataset("your_dataset")
recompute_stats(dataset, relative_action=True, chunk_size=50, relative_exclude_joints=["gripper"])
dataset.push_to_hub()
```

`chunk_size` 应该匹配你的策略的 `chunk_size`（π₀.₅ 默认为 50）。`relative_exclude_joints` 列出的关节名称应保持在绝对空间中（例如夹爪命令）。使用 `--push_to_hub true` 将更新的统计信息上传到 Hub。

然后启用相对动作进行训练：

```bash
lerobot-train \
    --dataset.repo_id=your_dataset \
    --policy.type=pi05 \
    --policy.use_relative_actions=true \
    --policy.relative_exclude_joints='["gripper"]' \
    ...
```

## 性能结果

### Libero 基准结果

π₀.₅ 在 Libero 基准套件上展示了强大的性能。为了比较和测试其 LeRobot 实现，我们在 Libero 数据集上对 libero 基础模型进行了额外的 6k 步微调，并将结果与 OpenPI 参考结果进行了比较。

| 基准 | LeRobot 实现 | OpenPI 参考 |
| ------------------ | ---------------------- | ---------------- |
| **Libero Spatial** | 97.0% | 98.8% |
| **Libero Object** | 99.0% | 98.2% |
| **Libero Goal** | 98.0% | 98.0% |
| **Libero 10** | 96.0% | 92.4% |
| **平均** | 97.5% | 96.85% |

这些结果展示了 π₀.₅ 在不同机器人操作任务中的强大泛化能力。要复现这些结果，你可以按照 [Libero](https://huggingface.co/docs/lerobot/libero) 部分中的说明进行操作。
