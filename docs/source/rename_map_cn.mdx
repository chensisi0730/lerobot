# 重命名映射与空相机

当你使用机器人策略进行训练、评估或录制时，你的**数据集**或**环境**在一组键下提供观测（例如 `observation.images.front`、`observation.images.eagle`），而你的**策略**期望另一组键（例如 `observation.images.image`、`observation.images.image2`）。**重命名映射**可以弥合这一差距，而无需更改策略或数据源。

> **范围：**重命名映射只重命名**观测**键（图像和状态）。动作键不受影响。

## 为什么观测键并不总是匹配

策略在其预训练配置中固定了一组**输入特征名称**。例如：

- [pi0fast-libero](https://huggingface.co/lerobot/pi0fast-libero) 期望 `observation.images.base_0_rgb` 和 `observation.images.left_wrist_0_rgb`。
- [xvla-base](https://huggingface.co/lerobot/xvla-base) 期望 `observation.images.image`、`observation.images.image2` 和 `observation.images.image3`。

你的数据集可能使用完全不同的名称（例如 `observation.images.front`、`observation.images.eagle`、`observation.images.glove`），而你的评估环境可能使用另一组名称。与其编辑策略配置或重命名数据集中的列，不如传递一个**重命名映射**：一个将源键映射到策略期望键的 JSON 字典。重命名在预处理器管道内进行，因此策略始终能看到它期望的键。

## 使用重命名映射

在命令行中将映射作为 JSON 字符串传递。约定始终是：

```
--rename_map='{"source_key": "policy_key", ...}'
```

其中 **source_key** 是数据集或环境提供的，而 **policy_key** 是策略期望的。

只有列出的键会被重命名；其他所有内容保持不变。条目的顺序无关紧要。

支持的策略：**PI0**、**PI05**、**PI0Fast**、**SmolVLA** 和 **XVLA**。

### 训练

假设你在一个数据集上微调 [lerobot/xvla-base](https://huggingface.co/lerobot/xvla-base)，该数据集的图像在 `observation.images.front`、`observation.images.eagle` 和 `observation.images.glove` 下。XVLA 期望 `observation.images.image`、`observation.images.image2` 和 `observation.images.image3`：

```bash
lerobot-train \
  --dataset.repo_id=YOUR_DATASET \
  --output_dir=./outputs/xvla_training \
  --job_name=xvla_training \
  --policy.path="lerobot/xvla-base" \
  --policy.repo_id="HF_USER/xvla-your-robot" \
  --policy.dtype=bfloat16 \
  --policy.action_mode=auto \
  --steps=20000 \
  --policy.device=cuda \
  --policy.freeze_vision_encoder=false \
  --policy.freeze_language_encoder=false \
  --policy.train_policy_transformer=true \
  --policy.train_soft_prompts=true \
  --rename_map='{"observation.images.front": "observation.images.image", "observation.images.eagle": "observation.images.image2", "observation.images.glove": "observation.images.image3"}'
```

### 评估

一个期望 `observation.images.base_0_rgb` 和 `observation.images.left_wrist_0_rgb` 的策略（例如 [pi0fast-libero](https://huggingface.co/lerobot/pi0fast-libero)），但 LIBERO 环境返回 `observation.images.image` 和 `observation.images.image2`：

```bash
lerobot-eval \
  --policy.path=lerobot/pi0fast-libero \
  --env.type=libero \
  ... \
  --rename_map='{"observation.images.image": "observation.images.base_0_rgb", "observation.images.image2": "observation.images.left_wrist_0_rgb"}'
```

## 替代方案：直接编辑策略配置

如果你始终使用相同的数据集或环境，你可以**编辑策略的 `config.json`**使其观测键与你的数据源匹配。然后就不需要重命名映射了。

权衡：修改策略配置会使它绑定到一个数据源。重命名映射可以使一个策略在多个数据集和环境之间通用。

## 空相机：比策略期望的视角更少

一些策略针对固定数量的图像输入而构建。如果你的数据集有更少的相机，你可以在策略配置中设置**`empty_cameras`**，而不是修改模型架构。

### 工作原理

设置 `empty_cameras=N` 会将 N 个占位符图像特征添加到策略配置中，命名为：

```
observation.images.empty_camera_0
observation.images.empty_camera_1
...
```

在运行时，这些键在批次中没有对应的数据。策略会用掩码虚拟张量填充它们（对于基于 SigLIP 的视觉编码器，用 `-1` 填充，用零注意力掩码），因此额外的图像槽位在训练和推理期间会被有效忽略。

### 示例

XVLA-base 默认有三个视觉输入和 `empty_cameras=0`。你的数据集只有两个相机：

1. 设置 `--policy.empty_cameras=1`。
2. 配置添加第三个键：`observation.images.empty_camera_0`。
3. 像往常一样为你的两个真实相机使用重命名映射。
4. 第三个槽位被掩码——不需要在数据集中使用假图像。

## 快速参考

| 目标 | 做什么 |
| --------------------------------------- | --------------------------------------------------------------------------- |
| 数据集键 ≠ 策略键 | `--rename_map='{"dataset_key": "policy_key", ...}'` |
| 环境键 ≠ 策略键（评估）| `--rename_map='{"env_key": "policy_key", ...}'` |
| 使用不同键进行 rollout（推理）| `--rename_map='{"source_key": "policy_key", ...}'` |
| 比策略期望的相机更少 | `--policy.empty_cameras=N`（由 PI0、PI05、PI0Fast、SmolVLA、XVLA 支持）|
| 避免传递重命名映射 | 编辑策略的 `config.json` 使其键与你的数据源匹配 |
