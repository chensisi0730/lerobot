## 设置你的环境

1. 按照我们的[安装指南](./installation)安装 LeRobot。
2. 通过运行以下命令安装 SmolVLA 依赖：

   ```bash
   pip install -e ".[smolvla]"
   ```

## 收集数据集

SmolVLA 是一个基础模型，因此需要在你的数据上进行微调以获得设置中的最佳性能。
我们建议从录制约 50 个 episode 的任务开始。按照我们的指南入门：[录制数据集](./il_robots)

<Tip>

在你的数据集中，确保每个变体（例如立方体拾放任务中桌子上立方体的位置）有足够的演示次数。

我们建议查看下面链接的数据集作为参考，该数据集用于 [SmolVLA 论文](https://huggingface.co/papers/2506.01844)：

🔗 [SVLA SO100 PickPlace](https://huggingface.co/spaces/lerobot/visualize_dataset?path=%2Flerobot%2Fsvla_so100_pickplace%2Fepisode_0)

在这个数据集中，我们在 5 个不同的立方体位置录制了 50 个 episode。对于每个位置，我们收集了 10 个拾放交互 episode。这种结构，重复每个变体多次，帮助模型更好地泛化。我们尝试了 25 个 episode 的类似数据集，但不够，导致性能不佳。所以，数据质量和数量绝对是关键。
在你的数据集在 Hub 上可用后，你可以使用我们的微调脚本将 SmolVLA 适配到你的应用。

</Tip>

## 在你的数据上微调 SmolVLA

使用我们的预训练 450M 模型 [`smolvla_base`](https://hf.co/lerobot/smolvla_base)，并在你的数据上进行微调。
在单个 A100 GPU 上训练模型 20k 步大约需要 4 小时。你应该根据性能和使用情况调整步数。

如果你没有 GPU 设备，你可以使用我们的笔记本在 [![Google Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/huggingface/notebooks/blob/main/lerobot/training-smolvla.ipynb) 上训练。

使用 `--dataset.repo_id` 将你的数据集传递给训练脚本。如果你想要测试安装，运行以下命令，我们使用为 [SmolVLA 论文](https://huggingface.co/papers/2506.01844) 收集的数据集之一。

```bash
cd lerobot && lerobot-train \
  --policy.path=lerobot/smolvla_base \
  --dataset.repo_id=${HF_USER}/mydataset \
  --batch_size=64 \
  --steps=20000 \
  --output_dir=outputs/train/my_smolvla \
  --job_name=my_smolvla_training \
  --policy.device=cuda \
  --wandb.enable=true
```

<Tip>
  你可以从小的批量大小开始，如果 GPU 允许的话，随着加载时间保持较短，可以逐步增加它。
</Tip>

微调是一门艺术。要获得微调选项的完整概述，运行

```bash
lerobot-train --help
```

## 评估微调后的模型并实时运行

与录制 episode 时类似，建议你登录 HuggingFace Hub。你可以按照相应的步骤操作：[录制数据集](./il_robots)
登录后，你可以通过以下命令在你的设置中运行推理：

```bash
lerobot-record \
  --robot.type=so101_follower \
  --robot.port=/dev/ttyACM0 \ # <- 使用你的端口
  --robot.id=my_blue_follower_arm \ # <- 使用你的机器人 id
  --robot.cameras="{ front: {type: opencv, index_or_path: 8, width: 640, height: 480, fps: 30}}" \ # <- 使用你的相机
  --dataset.single_task="Grasp a lego block and put it in the bin." \ <- 使用与数据集录制中相同的任务描述
  --dataset.repo_id=${HF_USER}/eval_DATASET_NAME_test \  # <- 这将是 HF Hub 上的数据集名称
  --dataset.episode_time_s=50 \
  --dataset.num_episodes=10 \
  --dataset.streaming_encoding=true \
  --dataset.encoder_threads=2 \
  # --dataset.vcodec=auto \
  # <- 如果你想在 episode 之间进行遥操作，遥操作是可选的
  # --teleop.type=so100_leader \
  # --teleop.port=/dev/ttyACM0 \
  # --teleop.id=my_red_leader_arm \
  --policy.path=HF_USER/FINETUNE_MODEL_NAME # <- 使用你的微调模型
```

根据你的评估设置，你可以配置录制评估套件的持续时间和 episode 数量。
