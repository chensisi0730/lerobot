# WALL-OSS

WALL-OSS 是一个用于具身智能的开源基础模型，由 [XSquare Robot](https://x2robot.com/en/research/68bc2cde8497d7f238dde690) 团队于 2025 年提出。LeRobot 实现改编自他们的开源 [WallX](https://github.com/X-Square-Robot/wall-x) 仓库。

X Square Robot 的 WALL-OSS 现已集成到 Hugging Face 的 LeRobot 生态系统中。这是 LeRobot 和 X Square Robot 团队之间的一个令人兴奋的合作项目。你现在可以直接通过 LeRobot 对 WALL-OSS 进行微调、评估和部署。通过这一点，我们的目标是让开源机器人社区更容易定制和部署 WALL-OSS 基础模型。阅读并探索 WALL-OSS [论文](https://arxiv.org/pdf/2509.11766) 和 [代码](https://github.com/X-Square-Robot/wall-x)。

## 模型概述

WALL-OSS 团队正在构建具身基础模型，以捕获和压缩世界上最宝贵的数据：物理交互的连续、高保真流。通过在模型的决策和身体的真实体验之间创建直接反馈循环，真正可泛化智能的出现成为可能——这种智能不仅理解世界如何运作，还理解如何在世界中有效行动。

从技术上讲，WALL-OSS 引入了一个紧密耦合的多模态架构（紧密耦合的 MoE 结构），整合了离散和连续动作建模策略。通过两阶段训练管道（Inspiration → Integration），模型逐渐统一了语义推理和高频动作生成。其核心创新包括：

- **具身感知增强的多模态预训练**：在统一的视觉-语言-动作数据上进行大规模训练，以加强空间、因果和操作理解。
- **统一跨层级思维链 (Uni-CoT)**：一个单一的可微框架，统一高层指令推理、子任务分解和细粒度动作合成，形成从"理解"到"执行"的连续链。
- **专家混合 (MoE) 动作头**：根据任务阶段动态激活专家，并在离散或连续空间中建模动作，以保持稳定的 VLM 先验。
- **两阶段训练范式**：
  - **Inspiration 阶段**：注入离散动作先验以加强空间理解和语义-动作对齐。
  - **Integration 阶段**：使用流匹配实现高频连续控制。

## 安装要求

1. 按照我们的[安装指南](./installation)安装 LeRobot。
2. 通过运行以下命令安装 WallX 依赖：

   ```bash
   pip install -e ".[wallx]"
   ```

## 使用方法

要在 LeRobot 中使用 WallX，请指定策略类型为：

```python
policy.type=wall_x
```

## 训练

对于训练 WallX，你可以使用带有适当配置的标准 LeRobot 训练脚本：

```bash
lerobot-train \
    --dataset.repo_id=your_dataset \
    --policy.type=wall_x \
    --output_dir=./outputs/wallx_training \
    --job_name=wallx_training \
    --policy.repo_id=your_repo_id \
    --policy.pretrained_name_or_path=x-square-robot/wall-oss-flow \
    --policy.prediction_mode=diffusion \
    --policy.attn_implementation=eager \
    --steps=3000 \
    --policy.device=cuda \
    --batch_size=32
```

### 训练参数

| 参数 | 描述 |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--dataset.repo_id` | 你训练数据集的 Hugging Face Hub 仓库 ID（例如 `lerobot/aloha_sim_insertion_human`） |
| `--policy.type` | 指定使用 WallX 策略架构 |
| `--output_dir` | 本地目录，用于保存训练检查点和日志 |
| `--job_name` | 此次训练运行的名称标识符（用于日志/跟踪） |
| `--policy.repo_id` | 你训练模型将被推送到的 Hugging Face Hub 仓库 ID |
| `--policy.pretrained_path` | 用于初始化的预训练 WallX 权重的路径（官方 WALL-OSS 检查点）|
| `--policy.prediction_mode` | 动作预测策略：`diffusion` 或 `fast` - `diffusion` 使用迭代去噪进行动作生成，`fast` 使用下一个标记预测 |
| `--policy.attn_implementation` | 注意力实现后端 - `eager` 使用标准 PyTorch 注意力（替代方案包括 `flash_attention_2` 或 `sdpa`）|
| `--steps` | 要运行的训练步数总数 |
| `--policy.device` | 训练设备（GPU 用 `cuda`，CPU 用 `cpu`）|
| `--batch_size` | 每个训练批次的样本数 |
